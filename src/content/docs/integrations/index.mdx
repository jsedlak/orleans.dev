Orleans uses a **provider model** that lets you plug in different backends for clustering, grain persistence, reminders, streaming, grain directory, and transactions. You choose the technologies that fit your infrastructure and Orleans handles the wiring.

## Provider Capabilities

Each backend supports a different set of Orleans features. Use this matrix to find the right provider for your needs.

| [Azure Cosmos DB](/integrations/azure-cosmos-db/) | Yes | Yes | Yes | — | — | — |
| [Azure Event Hubs](/integrations/azure-event-hubs/) | — | — | — | Yes | — | — |

<Aside type="tip">
  For development and testing, Orleans includes in-memory providers for clustering, persistence, reminders, and streaming. These are built into `Microsoft.Orleans.Server` and require no additional packages. See the [Getting Started](/guides/getting-started/) guide for an example.
</Aside>

## Choosing a Provider

**Cloud-native on Azure** — [Azure Storage](/integrations/azure-storage/) covers clustering, persistence, reminders, streaming, and transactions in a single service. Pair it with [Azure Event Hubs](/integrations/azure-event-hubs/) for high-throughput streaming.

**Cloud-native on AWS** — Use [DynamoDB](/integrations/aws-dynamodb/) for clustering, persistence, and reminders, and [SQS](/integrations/aws-sqs/) for streaming.

**Redis everywhere** — [Redis](/integrations/redis/) is a strong choice when you want a single backend for clustering, persistence, reminders, and grain directory across any cloud or on-premises environment.

**Relational databases** — [ADO.NET](/integrations/ado-net/) supports SQL Server, PostgreSQL, MySQL, and MariaDB for clustering, persistence, and reminders.

**Event streaming** — [Azure Event Hubs](/integrations/azure-event-hubs/) for high-throughput partitioned streams, [Apache Kafka](/integrations/apache-kafka/) for on-premises or multi-cloud event infrastructure, and [AWS SQS](/integrations/aws-sqs/) or [Azure Queue Storage](/integrations/azure-storage/) for simpler queue-based streaming.
